## Connection Layer

Support three connection types to an ELM327 adapter:
- **Bluetooth** — paired device selection, reconnect on drop
- **Wi-Fi** — TCP socket to ELM327 Wi-Fi adapter (default 192.168.0.10:35000)
- **USB/OTG** — serial-over-USB via UsbManager

Abstract all three behind a single `OBD2Connection` interface so the rest of the app is connection-agnostic. Handle connection errors, timeouts, and retries gracefully with user-facing status messages.

## OBD2 Protocol Support

Implement full support for **all OBD2 protocols** via the ELM327 AT command set, including protocol auto-detection and manual selection:

- **SAE J1850 PWM** (41.6 kbaud) — Ford
- **SAE J1850 VPW** (10.4 kbaud) — GM/Chrysler
- **ISO 9141-2** (5 baud init, 10.4 kbaud) — European/Asian vehicles
- **ISO 14230-4 KWP** (5 baud init and fast init variants)
- **ISO 15765-4 CAN** — 11-bit/29-bit identifiers, 250 kbaud and 500 kbaud variants (the dominant modern protocol)
- **SAE J1939** (heavy-duty CAN)
- **ISO 15765-4 CAN extended** variants

Implement the full ELM327 AT initialization sequence:
1. `ATZ` — reset
2. `ATE0` — echo off
3. `ATL0` — linefeeds off
4. `ATH1` — headers on
5. `ATSP0` — auto-detect protocol (or `ATSPx` for manual)
6. Query `ATDP` / `ATDPN` to confirm detected protocol and display it in the Dashboard and Vehicle tabs

**Response parsing:** Handle all ELM327 response types — `OK`, `NO DATA`, `UNABLE TO CONNECT`, `BUS BUSY`, `BUS ERROR`, `CAN ERROR`, `DATA ERROR`, `<DATA ERROR`, `STOPPED`, `?` — and surface meaningful feedback to the user for each. Parse multi-line and multi-frame (ISO-TP) responses correctly. Strip headers, length bytes, and padding before decoding PID values. Handle both 11-bit (7E8/7E9) and 29-bit CAN response headers.

**Protocol-specific decoding:** Implement correct byte-order and scaling for each protocol. For CAN/ISO-TP, handle segmented responses (first frame, consecutive frame, flow control). Display the active protocol name and numeric code in the Dashboard connection status area and in Vehicle → OBD2 Module Info.

### DTCs

### Service

- Category label
- Required engine state (**Engine ON** / **Engine OFF**) where applicable
- Warning indicator where applicable
- RUN/execute action

Implement all commands shown in the screenshots:
- **Engine:** Oil Life Reset, Throttle Body Relearn (Engine OFF), Idle Speed Relearn (Engine ON), EVAP Monitor Reset, Injector Fuel Trim Reset, MAF Adaptation Reset

- **Brakes:** EPB Retract, EPB Apply, ABS Pump Bleeding, Brake Wear Indicator Reset
- **Steering:** Steering Angle Calibration, EPS Torque Sensor Reset (Engine ON)
- **Tires:** TPMS Sensor Relearn, TPMS Pressure Baseline
- **Body:** Window/Sunroof Calibration
- **Battery:** Battery Registration, Charging System Reset
- **HVAC:** HVAC Blend Door Calibration, A/C Service Reset
- **Keys & Security:** Add Key Fob/Remote, Relearn All Remotes, IMMO Relearn after BCM swap, All Keys Lost Recovery, Remote Start Module Relearn, Key Fob Button Tests, Passive Entry Antenna Calibration, RF/transponder key programming

### Bi-Di (Bi-Directional Controls)
